# Project Manual — Mini-Project -- Focus Sound Player

## Overview

| | |
|--|--|
| **Capstone label** | Mini-Project -- Focus Sound Player |
| **Application name** | Focus Sound Machine |
| **Purpose** | Relaxing noise and ambience mixer for calm focus |

The browser (`client/`) provides the UI and plays audio with the **Web Audio API**. The **FastAPI** backend (`server/`) validates user settings, serves the track catalog and MP3 files, and returns normalized playback parameters before and during playback.

| Layer | Technology |
|-------|------------|
| Client | HTML, CSS, JavaScript (web browser) |
| Server | Python, FastAPI, uvicorn |
| Generated audio | White noise and brown noise buffers in the browser |
| Ambience audio | MP3 loops served from `/audio` |

## Architecture

```
Browser (client/)                    FastAPI (server/)
     |                                      |
     |-- GET /api/tracks ------------------>|
     |<-- sound source catalog --------------|
     |                                      |
     |-- POST /api/session/config --------->|
     |    (source, tone, volume, pan,       |
     |     timer)                           |
     |<-- validated playback config ---------|
     |                                      |
     |-- GET /audio/*.mp3 ----------------->|
     |<-- audio bytes -----------------------|
     |                                      |
     |-- GET /credits ---------------------->|
     |<-- credits.html ---------------------|
```

### Audio signal chain (client)

```
Source (generated buffer or decoded MP3)
  → BiquadFilter (tone; lowpass for generated noise only)
  → Gain (volume)
  → StereoPanner (left/right balance)
  → Speakers
```

User input is collected in the browser and **processed on the server** via `POST /api/session/config`. The client applies server-approved values: `effective_gain`, `filter_type`, `filter_frequency_hz`, `pan_value`, and `tone_control_enabled`.

## User interface

### Main mixer (`/`)

- **Sound source:** Dropdown loaded from `GET /api/tracks`. Status shows `Ready` when idle and `Playing [source name]` during playback.
- **Tone:** Active only for **Generated White Noise** and **Generated Brown Noise**; greyed out and disabled for ambience sources.
- **Volume:** 0–100% (default 20%).
- **Pan:** −100 (left) to +100 (right), default center.
- **Timer:** Minutes before auto-stop; `0` means play until paused.

### Credits page (`/credits`)

Lists attributions for ambience audio (Pixabay) and notes that white and brown noise were **generated by Cursor AI** in code (not downloaded files).

## Sound sources (current catalog)

Defined in `server/config.py`:

1. Generated White Noise — browser-generated
2. Generated Brown Noise — browser-generated
3. Ocean Ambience — `ocean-ambience.mp3`
4. Rain Ambience — `rain-ambience.mp3`
5. Air Conditioning Unit — `air-conditioning-unit.mp3`
6. Cozy Fireplace — `cozy-fireplace.mp3`
7. Train Passingby — `train-passingby.mp3`

To add a source: add an entry to `TRACK_CATALOG`. For file-based audio, place the MP3 in `audio/` and set `file` to the filename. For generated noise, set `file: None` and add the id to `GENERATED_NOISE_IDS`.

## Running locally

From the project root:

```powershell
py -3 -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\uvicorn.exe server.main:app --reload --host 127.0.0.1 --port 8000
```

Open [http://127.0.0.1:8000](http://127.0.0.1:8000).

## API reference (summary)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/tracks` | Sound source catalog |
| POST | `/api/session/config` | Validate and normalize session settings |
| GET | `/audio/{file}` | Ambience MP3 files |
| GET | `/` | Main UI |
| GET | `/credits` | Credits page |

## Extending to the public web

Deploy uvicorn behind a reverse proxy or PaaS. The same client/server split works on the public internet with minimal changes; no change to the core API contract is required.
